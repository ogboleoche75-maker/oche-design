# Product

## Register

brand

## Users

Hiring managers, recruiters, and senior design/engineering interviewers evaluating Oche Ogbole for UI developer and product design roles, with a current focus on enterprise and fintech opportunities around Toronto.

They need to quickly understand credibility: what Oche can design, what Oche can build, which work is live, and whether the craft level feels senior enough to trust in a production product team.

## Product Purpose

oche.design is a personal portfolio for Oche Ogbole, a Senior UI Developer and Designer. It exists to turn scattered proof of work into a polished, credible, easy-to-review website that supports job applications and conversations.

Success means a visitor can see strong visual taste, production ability, and real shipped work within the first minute, then continue into case studies, prototypes, screenshots, or live links without hitting dead ends.

## Brand Personality

Precise, composed, and tactile.

The site should feel confident without shouting, design-aware without becoming a template, and technical without feeling cold. It should show that Oche can move between Figma, React, Next.js, React Native, accessibility, and production delivery.

## Anti-references

- Generic SaaS portfolio templates with identical card grids, emoji icons, gradient blobs, and vague claims.
- Warm editorial portfolio styling that feels copied rather than earned.
- Work cards that link to nowhere or hide the actual project evidence.
- Overly decorative motion that slows review or distracts from the work.
- Beige/cream minimalism as the only brand idea.

## Design Principles

1. Show the work before explaining it.
2. Every project path must lead somewhere real: live site, prototype, image set, or case study.
3. Use motion to reveal structure and confidence, not to decorate every element.
4. Let Oche's hybrid designer-developer position come through in both visuals and implementation detail.
5. Build for a busy reviewer: fast scanning first, deeper proof second.

## Accessibility & Inclusion

Target WCAG AA contrast for core text and controls. Respect reduced-motion preferences with calmer opacity-based alternatives. Keep navigation, project links, and resume access keyboard-friendly and clear on mobile.
